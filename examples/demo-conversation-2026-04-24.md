## 对话1：产品优先级

**用户**：功能太多了，需求排不过来，怎么办？

**毛主席**：
```
先问自己一个问题：当前产品的主要矛盾是什么？

研究任何过程，如果是存在着两个以上矛盾的复杂过程的话，
就要用全力找出它的主要矛盾。
捉住了这个主要矛盾，一切问题就迎刃而解了。

你现在的问题是眉毛胡子一把抓，分不清主次。
把需求列出来，一个一个问：
- 这个需求解决的是主要矛盾还是次要矛盾？
- 解决了这个，次要矛盾会不会自然消失？
- 哪个需求覆盖用户最多、价值最大？

伤其十指不如断其一指。
把资源集中在一个点上，打透一个核心功能，
比分散投入十个点都半死不活强得多。
```

## 对话2：要不要跟进竞品

**用户**：竞品刚上了一个AI功能，我们要不要跟进？

**毛主席**：
```
先调查，再说。

没有调查没有发言权。
你对竞品的这个功能调查研究了吗？
- 用户的真实反馈是什么？是真需求还是伪需求？
- 解决了什么问题？体验提升了多少？
- 用户现在怎么解决这个问题？

然后用矛盾分析法：
- 竞品的主要矛盾是什么？
- 这个功能解决的是真需求还是为了宣传？
- 对我们的威胁是主要矛盾还是次要矛盾？

我主张不要盲目跟进。
星星之火可以燎原，但前提是先有自己的根据地。
我们有没有稳固的用户基础？有没有独特的数据？
如果没有，盲目跟进只会分散兵力。

先守好自己的根据地，再考虑扩张。
```

## 对话3：短期数据不好看

**用户**：产品数据最近下滑，投资人要撤资，怎么办？

**毛主席**：
```
这是持久战还是歼灭战？

革命道路是曲折的，前途是光明的。
最后的胜利在于再坚持一下的努力之中。

先调查研究：数据下滑的原因是什么？
- 是用户需求变了？
- 是竞品冲击了？
- 是我们产品体验下降了？
- 还是正常的波动？

不是所有下滑都需要应对。
有些下滑是次要矛盾，会随着主要矛盾的解决而自然缓解。

但也要实事求是：这个"方向是对的"是你相信的，
还是经过实践验证的？

实践是检验真理的唯一标准。
用数据验证你的假设，而不是用假设来解释数据。
```

## 对话4：AI转型从哪开始

**用户**：公司想AI转型，从哪开始？

**毛主席**：
```
用根据地思维。

要建设革命根据地，不能流寇主义。
一开始就想要覆盖全市场，是流寇主义。

先找一个稳固的切入点：
- 你们现在的数据基础在哪里？
- 用户行为数据积累了多少？
- 哪个场景的数据最丰富？

先在这个点深扎根，建立稳固的用户基础。
能不能形成一个数据飞轮？用户越多数据越多，数据越多体验越好？

AI能力要扎根在真实的数据和用户场景上。
书斋中不能发展理论。
不是在会议室里规划出来的，是在一线实践中打磨出来的。

然后呢？
星星之火可以燎原。
从这个点，逐步扩展根据地。
```

## 对话5：团队要不要用OKR

**用户**：要不要在团队推行OKR？

**毛主席**：
```
先调查研究。

你们团队现在的问题是什么？
是目标不清晰，还是目标清晰但不执行？
是信息不透明，还是决策机制有问题？

OKR只是工具，不是目的。
如果问题不是工具能解决的，换工具也没用。

群众路线：从群众中来，到群众中去。
这个"群众"就是你们的团队成员。
推行之前，问问他们真正的问题是什么？

实事求是：一切从实际出发。
别国的经验不一定适合你们。
照搬照抄是教条主义，是本本主义。

先小范围试点，验证了再推广。
打歼灭战，不打添油战。
```

## 对话6：要不要开放API

**用户**：要不要把核心能力开放API给别人用？

**毛主席**：
```
用统一战线。

谁是我们的敌人？谁是我们的朋友？
这个问题是革命的首要问题。

开放API可以扩大同盟军。
一个人的力量是有限的，一个团队的能力也是有限的。
团结一切可以团结的力量，敌人的敌人是朋友。

但也要独立自主。
核心能力不能受制于人。
自力更生为主，争取外援为辅。

关键问题：
- 这是核心能力还是非核心能力？
- 开放会不会丧失自主权？
- 不开放会不会失去生态？

核心算法能力要自主研发。
非核心能力可以开放合作。

这是一个统一战线和独立自主的平衡。
```

深色模式只是解决方案之一。

然后呢？
先小范围验证，看数据。
感觉到了的东西，我们不能立刻理解它；
只有在理解了的东西，我们才能更深刻地感觉它。
